## 四、技术架构

### 4.1 架构图

```
用户 → Nginx (80/443/8080)
          ↓
   Express Server (3001)
     ├── 静态文件 (dist/public/)
     ├── tRPC API (/api/)
     └── OAuth (已禁用)
```

### 4.2 技术栈

| 层次 | 技术 |
|------|------|
| **前端框架** | React 19 + TypeScript |
| **路由** | wouter（轻量级） |
| **状态管理** | React Query (TanStack) |
| **API** | tRPC（前后端类型共享） |
| **服务端** | Express 4 + esbuild 打包 |
| **构建工具** | Vite 7 |
| **数据库** | MySQL 8.0 + Drizzle ORM |
| **UI组件** | Radix UI + Tailwind CSS 4 |
| **部署** | PM2 + Nginx 反向代理 |

### 4.3 项目结构

```
/www/wwwroot/shipinzs/
  ├── client/          前端源码 (React SPA)
  │   ├── src/
  │   │   ├── pages/      页面组件
  │   │   ├── components/ UI组件
  │   │   ├── hooks/      自定义hooks
  │   │   └── lib/        工具函数
  │   └── public/        静态资源 (sitemap, robots.txt)
  ├── server/          后端源码 (Express + tRPC)
  │   ├── _core/       核心模块 (路由、OAuth、DB等)
  │   └── routers/     tRPC路由
  ├── shared/          前后端共享类型
  ├── drizzle/         DB schema + migrations
  └── dist/            构建产物
```
